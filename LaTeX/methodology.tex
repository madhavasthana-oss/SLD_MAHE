\subsection{System Overview}

The pipeline runs in two sequential stages at inference time:

\begin{enumerate}
    \item \textbf{Hand Detection (MediaPipe):} Each video frame goes into a MediaPipe model, which outputs bounding boxes around any detected hands.
    \item \textbf{Letter Classification (ResNet-50):} The detected hand crop gets extracted, resized, and fed to the classifier. It returns a probability distribution over 26 ASL letter classes.
\end{enumerate}

The predicted letter and bounding box are overlaid on the frame for visual feedback.

\subsection{Hand Detection Module}

The \texttt{HandDetector} class wraps MediaPipe's HandLandmarker Tasks API. We run it in 
\texttt{VIDEO} mode so it can do frame-to-frame tracking using MediaPipe's internal Kalman 
filter. One thing this mode requires: timestamps have to be strictly monotonically increasing, 
in real wall-clock milliseconds. A lot of implementations just pass integer counters (1, 2, 3...) 
which breaks the Kalman filter's velocity model and causes landmark jitter. We use 
\texttt{time.monotonic\_ns()} to avoid that.

Detection thresholds are set higher than MediaPipe's defaults. Hand detection and presence 
confidence at 0.65 (default is 0.50), tracking at 0.55. The defaults are too loose for 
static-sign ASL — partially occluded hands during transitions cause too many false 
detections otherwise.

Each frame, the detector returns up to two hand results. For each hand, the raw bounding 
box comes from the min/max of all 21 landmark coordinates, then expanded 35\% in each 
dimension (\texttt{expansion=0.35}) to give some context around the edges. That box then 
gets smoothed across frames using exponential smoothing: $0.3 \cdot \text{prev} + 0.7 \cdot \text{new}$.

Motion magnitude is computed on the \emph{raw} (pre-smoothing) center against the previous 
smoothed center — not the other way around. If you compute motion after smoothing, a fast-moving 
hand looks stable because the smoothed center barely moves. That defeats the whole stability 
gate. Motion uses Euclidean (L2) distance; L1 underestimates diagonal motion by around 30\%.

Tracking state for each hand — previous bbox and center — is stored by hand index, not 
as one shared variable. A shared variable breaks silently once two hands appear. When a 
hand disappears, its state gets evicted so the next detection starts fresh. If the clamped 
ROI has zero area after boundary clamping, that detection is skipped entirely.

When multiple hands are detected at inference time, we take the one with the highest 
handedness classifier score:

\begin{equation}
    \text{det}^* = \arg\max_{d \in \mathcal{D}}\; d.\text{score}
\end{equation}

The \texttt{HandDetection} dataclass per hand carries: smoothed bbox $(x_1, y_1, x_2, y_2)$, 
handedness label and score, 21 raw normalized landmarks, the BGR ROI crop, a stability flag, 
and raw motion magnitude in pixels.

\subsection{Classification Model (SLD)}

The Sign Language Detector (\textbf{SLD}) uses a frozen ResNet-50 backbone plus a trainable 
classification head. The backbone's average pooling gives a 2048-dimensional feature vector. 
We replace the original FC layer entirely. The head is:

\begin{equation}
    \hat{y} = \text{head}(\text{backbone}(x))
\end{equation}

\begin{equation}
    \text{head}(z) = W_2 \cdot \text{GELU}(\text{Dropout}(\text{LayerNorm}(z) \cdot W_1))
\end{equation}

where $W_1 \in \mathbb{R}^{2048 \times 512}$ and $W_2 \in \mathbb{R}^{512 \times 29}$, 
covering all 29 output classes (26 letters plus \texttt{del}, \texttt{nothing}, \texttt{space}).

We use \textbf{LayerNorm} at the backbone-to-head boundary instead of BatchNorm. BatchNorm 
is batch-size sensitive and behaves differently at train vs.\ eval time — that gets unstable 
when only the head is being updated. LayerNorm normalizes per sample so batch size doesn't 
matter. \textbf{GELU} is used over ReLU because it's smooth and has nonzero gradients at 
negative values. \textbf{Dropout} at $p = 0.3$ regularizes the 512-dimensional intermediate.

At inference, the checkpoint is loaded with \texttt{weights\_only=False}. This is intentional 
— the checkpoint contains optimizer and scheduler state in addition to model weights, so 
weights-only mode can't load it. The flag also suppresses the PyTorch 2.x deprecation warning.

\subsection{Training Configuration}

Training config lives in a \texttt{TRAINING\_CONFIG} dataclass. All hyperparameters are 
in one place, and command-line args can override them at runtime. Table~\ref{tab:config} 
has the key settings.

The data split uses \texttt{random\_split}: 70\% train, 20\% val, 10\% test. Transforms 
are applied per-split via a \texttt{\_TransformSubset} wrapper. Overwriting 
\texttt{subset.dataset} directly causes an \texttt{IndexError} — the Subset's indices 
point into the wrong-sized dataset. Wrapping the Subset itself avoids this. Training 
images get augmentation; val and test get only deterministic resize and normalization.

Per-batch metrics — loss, accuracy, gradient norm — are logged to W\&B at every step 
via \texttt{wandb.log(..., step=self.global\_step)}. At the epoch level we also log mean 
gradient norm and gradient clipping ratio (how often the norm exceeded the clip threshold). 
That gives a direct read on how hard the optimizer is being constrained.

\subsubsection{Learning Rate Schedule}

Cosine annealing with a linear warm-up. During warm-up (epochs $0$ to $T_w - 1$), 
learning rate goes up linearly:

\begin{equation}
    \eta_t = \eta_{\text{base}} \cdot \frac{t+1}{T_w}
\end{equation}

After that, cosine decay kicks in:

\begin{equation}
    \eta_t = \eta_{\min} + \frac{\eta_{\text{base}} - \eta_{\min}}{2} \left(1 + \cos\left(\pi \cdot \frac{t - T_w}{T_{\max} - T_w}\right)\right)
\end{equation}

Warm-up prevents instability early on; cosine decay settles the LR smoothly near convergence.

\subsubsection{Gradient Clipping}

Gradient norm clipping at $\tau = 1.0$ after each backward pass:

\begin{equation}
    \mathbf{g} \leftarrow \mathbf{g} \cdot \min\!\left(1,\ \frac{\tau}{\|\mathbf{g}\|_2}\right)
\end{equation}

\subsubsection{Early Stopping}

Training stops if val accuracy doesn't improve for 20 consecutive epochs. Best checkpoint 
— model, optimizer, scheduler, best accuracy — is saved whenever a new peak is hit. 
That checkpoint gets reloaded before final test evaluation, so test metrics reflect the 
best generalization point, not just the last epoch.

\subsection{Experiment Tracking}

All runs logged via W\&B \cite{wandb}. Per-batch: loss, accuracy, gradient norm. 
Per-epoch: val loss and accuracy, learning rate, mean gradient norm, clipping ratio. 
Final test metrics go in as summary values. The full \texttt{TRAINING\_CONFIG} — 
optimizer kwargs, scheduler kwargs, dataloader configs — gets serialized into the 
W\&B run config so runs are exactly reproducible.
