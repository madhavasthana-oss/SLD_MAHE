At the time of scanning, frames are captured online via a live webcam feed and passed into a classification
pipeline that follows this structure
\begin{enumerate}
    \item \textbf{Hand Detection:} Media pipe detects the hand and contextual environment.
    \item \textbf{Stability Check:} Euclidean motion between consecutive frames is used to flag instability
    \item \textbf{Crop and Pad:} This is invisible to the user, the box that establishes context is not visible.
    However, the skeleton tracing the landmarks of the detected hand is available to the user.
    \item \textbf{Classification:} The crop is resized to $224 \times 224$ and normalized. 
    It is passed through the SLD model which predicts several possible 
    classifications. A prediction is accepted only if confidence exceeds 0.50.
    \item \textbf{Temporal Voting:} A majority-vote buffer of size 10 is maintained. 
    A letter is emitted only when 70\% or more of the buffer agrees. 
    \item \textbf{Cooldown:} A 1-second cooldown is enforced. 
    The same letter cannot be emitted repeatedly in rapid succession.
\end{enumerate}

\subsection{FastAPI Inference Server}

The Inference Pipeline is available through a lightweight FastAPI \cite{fastapi} server as a REST endpoint.
Any HTTP client can send JPEG images to the server, while JSON output will be received.

\textbf{Endpoint:} \texttt{POST /predict} accepts a multipart-encoded JPEG image. The response carries:

\begin{itemize}
    \item \texttt{letter} — stable, voted prediction (A--Z, \texttt{del}, \texttt{nothing}, \texttt{space}). \texttt{null} if not yet stable.
    \item \texttt{confidence} — softmax probability of the top class.
    \item \texttt{bbox} — bounding box as $[x, y, w, h]$ in pixel coordinates.
    \item \texttt{handedness} — left or right, from MediaPipe.
    \item \texttt{motion} — Euclidean motion magnitude in pixels. Used for stability gating.
    \item \texttt{landmarks} — 21 normalized hand landmark coordinates. Used for frontend skeleton rendering.
    \item \texttt{error} — status string (\texttt{no hand}, \texttt{hand moving}, \texttt{low confidence}, \texttt{not stable yet}, \texttt{cooldown}) when no letter is emitted.
\end{itemize}

The Global State consists of several locks, namely \texttt{threading.Lock}, including those related to the Prediction Buffer, Last Emited Letter, and Cooldown Time Stamp. CORS is enabled for all origins.

\subsection{Browser Frontend}

Single-page HTML/JS frontend, connects to the FastAPI server. 
Client-side MediaPipe runs through the Tasks Vision JS SDK at 60\,fps -- for the skeleton 
overlay, which renders with zero latency. 
The actual classification frames go to the backend at around 8\,fps.

For letter emission we use a dwell-based approach. 
A letter has to stay stable for 1.8\,s before it gets added to the transcript. 
If the same letter repeats, it needs 3.6\,s. This way brief hand holds don't accidentally trigger output.
