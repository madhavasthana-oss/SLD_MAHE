\begin{abstract}
American Sign Language (ASL) is the main communication mode for the hearing-impaired community. 
Bridging the gap between deaf and hearing individuals through automated real-time recognition 
remains an open engineering challenge. This paper describes a two-stage pipeline we built: 
hand region localization via MediaPipe, then letter classification using a fine-tuned ResNet-50 
with a custom head. Training used the public ASL Alphabet dataset from Kaggle, evaluated over 
26 letter classes. We used cosine annealing with linear warm-up, gradient clipping, and early 
stopping to keep training stable. Final test accuracy was 99.80\%. The system runs live via 
webcam with skeleton overlays and letter labels on each frame. A FastAPI server handles inference; 
a browser frontend connects to it for interactive use. Overall the results show landmark-based 
detection combined with CNN classification is a viable path for low-latency ASL recognition.
\end{abstract}

\section{Related Work}

Early work on sign language recognition mostly used sensor-based setups —
data gloves, depth cameras — to get 3D hand pose data \cite{mitra2007}. 
Results were decent, but the hardware requirements made real deployment pretty impractical.

Deep learning shifted things toward image-based approaches. CNNs trained on static hand 
images got much better at classifying isolated gestures \cite{pigou2015}. Transfer learning 
from ImageNet-pretrained models like VGG-16, ResNet, and MobileNet helped a lot, especially 
on smaller domain-specific datasets where training from scratch wasn't an option.

Landmark-based tracking has been explored for gesture spotting. But a full two-stage 
system that handles the entire ASL alphabet in a live-camera setting — that's still 
relatively unexplored. This work tries to fill that gap with something actually deployable.

\section{Dataset}

We used the ASL Alphabet dataset from Kaggle \cite{kaggle_asl}. It has 87,000 labeled images 
across 29 classes: 26 letters (A--Z) plus \texttt{SPACE}, \texttt{DELETE}, and \texttt{NOTHING}. 
Images are $200 \times 200$ pixels, RGB.

All images were resized to $224 \times 224$ to match ResNet-50's expected input, then 
normalized with the standard ImageNet mean and std. We split the data into 
training (70\%), validation (20\%), and test (10\%) using random splitting. A 
\texttt{\_TransformSubset} wrapper handles the different transforms per split — training 
gets random horizontal flips and color jitter, validation and test just get deterministic 
resize and normalize. Batch size was 128; shuffling on for training, off for evaluation.

Split sizes came out to roughly 60,900 train, 17,400 val, 8,700 test.

\section{Conclusion}
We built a real-time ASL recognition system using MediaPipe for hand detection and ResNet-50 
for classification. Test accuracy hit 99.80\% on the Kaggle dataset. The system runs at 
interactive frame rates through a FastAPI server with a browser frontend. Training used 
warm-up scheduling, Nesterov SGD, gradient clipping, and early stopping. The temporal 
voting and dwell-based emission kept output stable in live use. There is real room to grow 
here — word-level ASL, continuous signing, better generalization to real-world hands.

Future directions:
\begin{itemize}
    \item Fine-tune MediaPipe on a dedicated hand detection dataset for better localization.
    \item Swap the backbone to ResNet-152 or EfficientNet for more classification capacity.
    \item Extend to dynamic gestures and full ASL words using LSTMs or Transformers.
    \item Package as a mobile app for wider reach.
\end{itemize}
